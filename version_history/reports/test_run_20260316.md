## Crash Cause ŌĆö RESOLVED

**Root cause:** Missing `anarchism` ideology subtype in `common/ideologies/AOTA_ideologies.txt`

AOTA promotes `anarchism` from a vanilla subtype (under `neutrality`) to a top-level ideology GROUP. However, the `anarchism` GROUP's `types = {}` block contained only sub-flavour types (`anarchist_communism`, `anarcho_syndicalist_subtype`, etc.) ŌĆö no self-named `anarchism` subtype.

Vanilla `common/national_focus/spain.txt:7988` calls:
```
create_country_leader = { ideology = anarchism ... }
```

`create_country_leader` requires an ideology **subtype** name. The engine attempted to look up an ideology type named `anarchism`, found none, returned a null pointer, and dereferenced it ŌåÆ EXCEPTION_ACCESS_VIOLATION ŌåÆ crash.

**Fix applied:** Added `anarchism = { can_be_randomly_selected = no }` as a type inside the anarchism ideology group. This matches vanilla's self-named subtype convention (communism group has `communism` type, fascism group has `fascism` type, etc.).

**Side effect fixed:** Eliminates the spam of ~100+ "Not a valid value: anarchism" errors from vanilla scripted localisation files (`FR_SCRIPTING_FULL_AUTOMATED.txt`, `00_scripted_localisation_countries_RU_loc.txt`, `NSB_NewsEvents.txt`) which use `has_country_leader_ideology = anarchism` ŌĆö now valid.

**File changed:** `common/ideologies/AOTA_ideologies.txt` (+1 line)
**Commit:** `6312fd8`

**`limit` used as trigger in `common/decisions/AOTA_china_unification_decisions.txt`**
Lines: 59, 93, 173, 204, 238
`limit` is not a valid standalone trigger ŌĆö it is only valid as a scope filter inside effects. Wherever these appear, they are likely intended to be inside an `available` or `visible` block. Requires careful inspection before fixing.

**`set_state_owner` invalid scope in `events/AOTA_events.txt`**
Lines: 173ŌĆō198, 233ŌĆō251
Pattern: `395 = { add_core_of = AWU set_state_owner = AWU }` ŌĆö the parser rejects `set_state_owner` in state scope in HOI4 1.17. These events fire during gameplay (not at load), so they do not cause crashes, but `set_state_owner` calls will silently fail when triggered. May need to be rewritten as `transfer_state` or called from country scope.

**`lse is not a valid tag of on_monthly` (onaction.cpp:461)**
The engine parses `on_monthly_pulse` as `on_monthly` + tag suffix `pulse`, then extracts `lse` (likely last 3 chars) as a tag ŌĆö invalid. `on_monthly_pulse` is not a valid HOI4 on_action keyword. Used in `common/on_actions/AOTA_china_japan_interaction_on_actions.txt`. The monthly logic inside this block is currently non-functional. Should be refactored to use `on_monthly` with an `every_country` limit instead.

## Fixes Applied (Session 2 ŌĆö 2026-03-16)

**`buddhist_socialism` missing ideology subtype**
Added `buddhist_socialism = {}` to AOTA's `communism` group types in `AOTA_ideologies.txt`. Vanilla `events/SEA_Japan.txt:11926` references this as an ideology subtype. Since AOTA uses `replace_path = "common/ideologies"`, this subtype was missing. Fix prevents potential null pointer crashes if vanilla Japan event content fires.

**`limit` invalid trigger in `common/decisions/AOTA_china_unification_decisions.txt`**
Fixed all 5 instances (lines 59, 93, 173, 204, 238). Pattern was `any_country = { limit = { ... } }` inside `available` trigger blocks. In HOI4 trigger context, `any_country` takes conditions directly without a `limit` wrapper. Removed the `limit = { }` wrapper from each instance.

**`set_state_owner` wrong scope in `events/AOTA_events.txt`**
Fixed in events aota.201 and aota.202. Replaced `STATE_ID = { ... set_state_owner = TAG }` (invalid in state scope) with `TAG = { transfer_state = STATE_ID ... }` in country scope. `add_core_of` remains in state scope (still valid there).

**`on_monthly_pulse` invalid on_action keyword**
Fixed `common/on_actions/AOTA_china_japan_interaction_on_actions.txt`. Renamed `on_monthly_pulse` to `on_monthly`. The `every_country` limit blocks inside already filter correctly, so the logic is preserved. China/Japan monthly system is now functional.
